## ขอบเขตการดำเนินการ เพื่อเชื่อมโยงบริการของหน่วยงานกับแพลตฟอร์มกลาง

** [ตัวอย่าง - หน่วยงานอาจปรับเปลี่ยนได้ตามความเหมาะสม] ** 

1. ศึกษารายละเอียดแพลตฟอร์มกลางของงานบริการภาครัฐสำหรับภาคธุรกิจและประชาชน ซึ่งครอบคลุมทั้ง ระบบพอร์ทัลกลางเพื่อประชาชน (แอปพลิเคชัน “ทางรัฐ”) และระบบศูนย์กลางบริการภาครัฐเพื่อภาคธุรกิจ (Biz Portal) จากเว็บไซต์ kb.dga.or.th ของสำนักงานพัฒนารัฐบาลดิจิทัล (องค์การมหาชน) (สพร.) หรือแหล่งอื่น ๆ ตามที่ผู้ว่าจ้างกำหนด
2. ออกแบบแนวทางการเชื่อมโยงบริการของหน่วยงานกับแพลตฟอร์มกลางของงานบริการภาครัฐ ฯ สำหรับภาคธุรกิจและประชาชน
3. พัฒนาบริการของหน่วยงานสำหรับเชื่อมโยงกับแพลตฟอร์มงานบริการภาครัฐสำหรับภาคธุรกิจและประชาชน (Service MiniApp) ซึ่งมีคุณสมบัติอย่างน้อย ดังนี้ [หน่วยงานอาจพิจารณาเพิ่ม หรือตัดคุณสมบัติบางประการได้ตามความเหมาะสม]
   - ปรับปรุงการแสดงผลให้เป็นแบบ responsive design หรือรูปแบบตามที่ สพร. หรือหน่วยงานกำหนด
   - เชื่อมโยงข้อมูลผู้ใช้งาน (User Profile) จากแพลตฟอร์มกลางของงานบริการภาครัฐ​ ฯ
   - ประชาชนสามารถดูข้อมูลของตนที่เกี่ยวข้องกับงานบริการของหน่วยงานได้ เช่น รายการ/และรายละเอียดใบอนุญาตที่ได้รับ (My Information)
   - ประชาชนสามารถกรอกรายละเอียดเพื่อขอรับบริการผ่าน e-Form ได้ โดยนำข้อมูลผู้ใช้งานจากแพลตฟอร์มกลาง ฯ ไปกรอก (Prefill) ไม่ต้องให้ประชาชนกรอกข้อมูลซ้ำซ้อน (Service Request)
   - ประชาชนสามารถติดตามสถานะความคืบหน้าการขอรับบริการได้ (Service Tracking)
   - แจ้งเตือนข้อมูลต่าง ๆ ให้ประชาชนทราบผ่านแพลตฟอร์มกลาง ฯ ได้ เช่น มีใบอนุญาตที่กำลังจะหมดอายุ หรือ การขอรับบริการที่ยื่นไว้ได้รับการอนุมัติแล้ว เป็นต้น (Service Notification)
4. ทดสอบบริการของหน่วยงานสำหรับเชื่อมโยงกับแพลตฟอร์มงานบริการภาครัฐสำหรับภาคธุรกิจและประชาชน (Service MiniApp) ซึ่งครอบคลุมถึง
การทดสอบการทำงานของบริการ ฯ ทั้งในแง่ความสามารถของบริการ (Functional Test), ประสิทธิภาพของบริการ (Performance Test), ความปลอดภัยของบริการ (Security Test) เป็นต้น
   - การทดสอบการเชื่อมต่อกับแพลตฟอร์มงานบริการภาครัฐ​ ฯ ร่วมกับ สพร. หรือ หน่วยงานที่เกี่ยวข้อง
5. ติดตั้งบริการของหน่วยงานสำหรับเชื่อมโยงกับแพลตฟอร์มงานบริการภาครัฐสำหรับภาคธุรกิจและประชาชน (Service MiniApp) ในสภาวะแวดล้อม และวิธีการที่ผู้ว่าจ้างกำหนด
6. จัดหา และติดตั้ง บริการของหน่วยงานให้อยู่ใน Server ที่มีความปลอดภัย โดยบริการที่พัฒนาขึ้น ต้องรองรับ SSL Certificate 
7. เตรียมข้อมูลที่เกี่ยวข้อง เพื่อนำบริการขึ้นแอปทางรัฐ เช่น ภาพ Icon, Screenshot, คำอธิบายบริการแบบย่อ, คำอธิบายบริการแบบเต็ม และอื่นๆที่เกี่ยวข้อง

## งวดงาน และงบประมาณ เพื่อเชื่อมโยงบริการของหน่วยงานกับแพลตฟอร์มกลาง

** [ตัวอย่าง - หน่วยงานอาจปรับเปลี่ยนได้ตามความเหมาะสม] ** 

1. งวดที่ 1 
   - แผนการดำเนินงานตลอดโครงการ
   - ผลการศึกษา และออกแบบแนวทางการเชื่อมโยงบริการของหน่วยงานกับแพลตฟอร์มกลางของงานบริการภาครัฐ ฯ สำหรับภาคธุรกิจและประชาชน ซึ่งครอบคลุมถึง
      - ขั้นตอนการทำงานของการใช้บริการผ่านแพลตฟอร์มงานบริการภาครัฐ ฯ (User Journey)
      - การออกแบบหน้าจอบริการของหน่วยงานสำหรับเชื่อมโยงกับแพลตฟอร์มงานบริการภาครัฐ ฯ (Service MiniApp)
      - รายละเอียดด้านเทคนิคที่ต้องปรับปรุง (Gap Analysis)
2. งวดที่ 2 บริการของหน่วยงานสำหรับเชื่อมโยงกับแพลตฟอร์มงานบริการภาครัฐสำหรับภาคธุรกิจและประชาชน (Service MiniApp) พร้อม Source Code และเอกสารผลการทดสอบระบบ พร้อมสำหรับการทดสอบระบบจากมุมมองผู้ใช้บริการ (User Acceptance Test)
3. งวดที่ 3
   - บริการของหน่วยงานสำหรับเชื่อมโยงกับแพลตฟอร์มงานบริการภาครัฐสำหรับภาคธุรกิจและประชาชน (Service MiniApp) พร้อม Source Code และเอกสารผลการทดสอบระบบ พร้อมสำหรับเปิดให้บริการจริง (Pre-Production)
   - รายงานผลการติดตั้งระบบ
   - รายงานการฝึกอบรม

## งบประมาณ

DGA เคยจ้างปรับปรุงบริการที่มีอยู่แล้วของหน่วยงานให้สามารถเชื่อมต่อกับแพลตฟอร์มกลาง ฯ โดยมีคุณสมบัติครบตามขอบเขตข้อ 3 อยู่ที่ประมาณ 5 แสนบาท แต่หากหน่วยงานจำเป็นต้องพัฒนาคุณสมบัติบางส่วนเพิ่มเติม หรือพัฒนาบริการขึ้นใหม่เลย ก็อาจใช้งบประมาณสูงกว่าที่ สพร. เคยดำเนินการได้
